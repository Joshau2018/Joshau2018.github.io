Step Four in Planning for Making an AI

If you decide that you need an API so your AI and user interface can communicate with each other, then how are you going to make an API?

If you’re using Python, then you might use Flask or FastAPI.

Flask is good for small to medium-sized projects. So, if you plan on building a simple demo of an AI, then Flask might be the best choice. If you wish to create a large-scale AI, then FastAPI might be better due to its speed and automatic data validation. Although FastAPI is faster, it’s newer, so it may be more difficult to find support if you’re a beginner. Flask, on the other hand, is older and has a larger community, making it easier for beginners to learn.

Use Case Differences

  • Flask: Better if you are building a website using HTML (turing.com).
  • FastAPI: Better if you plan on using React/Vue or a separate app for the frontend (turing.com).

Speed

  • Flask: Slower than FastAPI.
  • FastAPI: Faster than Flask.

Storage

  • Flask: Takes up slightly less space.
  • FastAPI: Only uses a couple more megabytes.

Community

  • Flask: Older and well-established community.
  • FastAPI: Newer, but actively growing community.

Complexity of Learning

  • Flask: Easier syntax, more beginner-friendly (geeksforgeeks.com).
  • FastAPI: Requires knowledge of asynchronous programming and Pydantic, making it harder to learn (geeksforgeeks.com).

Overall

Flask is an excellent choice if your AI is low-scale, uses HTML for the frontend, and you don’t have much experience with asynchronous programming. In contrast, FastAPI is better if you're willing to learn asynchronous programming and want to build a medium to large-scale AI.
